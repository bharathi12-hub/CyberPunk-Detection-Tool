            ⓘ
            About CyberPunk Detection Tool
          


          

            

              CyberPunk Detection Tool is a browser extension that automatically analyzes every
              page you visit and produces a single Trust Score (0–100), explaining exactly what
              it found and why, in plain English and in exportable reports.
            


            
How it works

            

              Every time a page finishes loading, the extension's background service worker runs
              a set of independent detection engines against that page — the URL, its DNS/registration
              record, its outgoing network requests, its response headers, its cookies, and its visible
              page content. Each engine returns a focused finding; a risk engine then combines all of
              them into one weighted score and a LOW / MEDIUM / HIGH RISK classification. Nothing is
              sent off your device for this scoring — it all happens locally in the extension. The only
              network calls made are to public threat-intelligence feeds (to check if a URL is already
              known-malicious) and, optionally, to your own self-hosted backend for AI-written
              explanations, scan history, and alert delivery.
            


            
What it scans for

            
    
              
  • Phishing & brand impersonation — homograph attacks (e.g. paypaI.com) and combo-squatting (e.g. paypal-secure-login.com)
    • 
              
  • Typosquatting — domains one character away from a well-known brand
    • 
              
  • Known-malicious URLs — cross-referenced against OpenPhish, PhishTank, and URLhaus (and AbuseIPDB / Google Safe Browsing / VirusTotal if your backend operator has configured those optional keys)
    • 
              
  • Crypto-scam language — phrasing commonly used in fake giveaway/airdrop scams
    • 
              
  • Trackers — known analytics/ad-tracking scripts loaded by the page
    • 
              
  • Browser fingerprinting — Canvas, WebGL, Audio, and font-enumeration fingerprint attempts
    • 
              
  • Insecure cookies — missing Secure/HttpOnly/SameSite attributes
    • 
              
  • Missing security headers — CSP, HSTS, X-Frame-Options, and related headers
    • 
              
  • Risky downloads — dangerous file types and double-extension tricks (e.g. invoice.pdf.exe)
    • 
              
  • Suspicious login forms & data leakage — forms that submit to a different domain than the page itself, and sensitive-data patterns (card numbers, API keys, etc.) found in page text
    • 
              
  • QR codes on the page — a real phishing vector ("quishing") that URL-based filters typically miss entirely
    • 
              
  • Excessive/repeated permission requests — camera, microphone, location, and notification prompts, especially repeated asks after a prior denial
    • 
            


            
How to operate it

            
    
              
  1. Browse normally — scanning happens automatically on every page load (toggle this off anytime in Settings → Scanning → Auto-scan)
    2. 
              
  2. Click the toolbar icon for an instant summary of the current page
    3. 
              
  3. Open the Dashboard for full scan history, search, and exportable reports (.txt / .pdf / .docx, single or bulk)
    4. 
              
  4. Use Settings (bottom-left of this sidebar) to choose a theme, dark/light mode, and notification preferences
    5. 
            


            
Limitations & honest notes

            

              This tool is a strong first-pass signal, not a guarantee. Brand-impersonation detection
              only checks against a fixed list of commonly-targeted brands; truly novel phishing
              campaigns it has never seen before may not be flagged by name-matching alone (the
              threat-feed and ML-signal checks help catch some of these, but no detector is complete).
              "Risk sensitivity" and "auto-block trackers" in Settings currently save your preference
              but are not yet behaviorally wired — this is disclosed plainly in Settings itself, not
              hidden. The Score is only for Reference not a Guarantee.
            


            
Who this suits

            

              Best suited for security-conscious individuals, students learning SOC/blue-team
              concepts, and anyone who wants a second opinion on a link before logging in or
              downloading something — not a replacement for enterprise endpoint protection,
              a corporate SIEM, or your own judgment about a site you already have reason to
              distrust.
